# 更新器下载进度手工验证清单

关联 Issue: `#104`

环境前提:
- Windows 10/11
- AutoHotkey v2
- 使用 `develop` 基线上的更新检查配置
- 更新源保持为 `CloudTracey/arknights-frame-assistant`

验证结果记录:
- [ ] 1. 正常检查更新并开始下载时，下载弹窗显示进度条、百分比和已下载大小，且数值持续变化。
- [ ] 2. 服务端未返回 `Content-Length` 时，进度条进入不确定模式，界面显示 `已下载 X`。
- [ ] 3. 点击“取消下载”后，按钮被禁用，文案切换为“正在取消下载...”，随后弹出“下载已取消”提示。
- [ ] 4. 取消后不会残留临时安装包、状态文件或 PowerShell worker 脚本。
- [ ] 5. 点击“手动下载”后会打开外部下载页面，并关闭当前下载弹窗。
- [ ] 6. 下载成功后仍会弹出下载完成提示，并继续进入现有自替换流程。
- [ ] 7. 下载失败或后台下载进程异常退出时，会沿用现有重试与失败提示逻辑。
- [ ] 8. 版本检查请求仍指向 `https://api.github.com/repos/CloudTracey/arknights-frame-assistant/releases`。

本地验证说明:
- [ ] 已完成上述手工验证，结果符合预期。
- [ ] 本次改动未引入 `version.ahk` 的 alpha 版本号变更。
- [ ] 本次改动未引入 fork 更新源或其他敏感信息。
